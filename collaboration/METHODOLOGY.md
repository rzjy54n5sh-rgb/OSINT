# MENA Intel Desk — Analytical Methodology

---

## Core Principle: Strict Neutrality

All analysis must present facts, probabilities, and scenarios without advocating for any actor, state, ideology, or outcome. The platform's credibility depends entirely on this.

**Neutrality rules:**
- Never frame any party as "right," "justified," "aggressor," or "defender"
- Never use morally loaded language (terrorist, freedom fighter, martyr, genocide — unless quoting a source)
- Always present multiple actor perspectives on any event
- Attribute claims to their source: "Iran state media reports...", "IDF spokesperson stated..."
- Distinguish clearly: confirmed fact vs. reported claim vs. analysis vs. speculation

---

## NAI Score (Neutrality Assessment Index)

Every report is self-assessed 0–100 for neutrality compliance before publishing.

| Score | Status |
|---|---|
| 85–100 | ✅ Publishable |
| 70–84 | ⚠️ Review required — identify which dimension failed |
| Below 70 | ❌ Rewrite required |

**NAI scoring dimensions (20 points each):**
1. **Source balance** — Multiple sides represented, not just one narrative
2. **Language neutrality** — No loaded terms, emotional framing, or implicit judgment
3. **Claim attribution** — Facts vs. claims vs. analysis clearly labeled throughout
4. **Scenario balance** — Scenarios weighted by evidence, not by preferred outcome
5. **Actor representation** — All relevant actors' stated positions included

---

## Scenario Framework

Every major development is analyzed through three scenarios:

| Scenario | Description |
|---|---|
| **S1 — De-escalation** | Diplomatic resolution, ceasefire, strategic pullback |
| **S2 — Stalemate** | Ongoing conflict, no decisive shift, frozen lines |
| **S3 — Escalation** | Widening conflict, new actors, strategic expansion |

Each scenario requires:
- **Probability %** (S1 + S2 + S3 must sum to 100)
- **Key trigger conditions** — what would materialize this scenario
- **Invalidation conditions** — what would rule it out
- **Regional impact** — Egypt, UAE, oil, Hormuz, Suez

Scenario probabilities are tracked over time in `SCENARIOS.md`.

## Source Hierarchy

See `SOURCES.md` for the full source registry. Summary:

- **Tier 1 (Primary):** Reuters, AP, AFP, BBC, Al Jazeera, ISW, ACLED, Crisis Group
- **Tier 2 (Secondary, use with attribution):** Sky News, PBS, Guardian, regional outlets
- **Tier 3 (Monitor only, never cite directly):** Telegram channels, social media, state-adjacent outlets (IRNA, PressTV, Times of Israel opinion)

---

## Report Structure Template

Every report follows this exact structure:

1. **Conflict State Summary** — Day X, theater overview, key developments last 24–72h
2. **Actor Postures** — Iran, Israel, US, proxies, Gulf states — stated positions
3. **Scenario Analysis** — S1/S2/S3 with current probabilities and trigger conditions
4. **Regional Impact** — Egypt-specific and UAE-specific consequences
5. **Business Intelligence** — Market/investment implications
6. **NAI Self-Assessment** — Score + which dimension, if any, needs attention
7. **Sources** — All sources used with links and tier classification

---

## Conflict Day System

Day 1 = February 28, 2026.  
Every report anchors to the current Conflict Day. This enables time-series tracking and comparison across analysts.

---

## Blind Analysis Protocol

Periodically, both collaborators independently analyze the same event without reading each other's output first. Then compare:
- Where you agree = high-confidence signal
- Where you diverge = investigate the cause (different sources? different priors? different framing?)

Blind analysis results are logged in `DEBATES.md` under the tag `[Blind]`.

---

## Language Standards

- Use probability ranges, not certainties: "60–70% probability" not "will happen"
- Acknowledge uncertainty explicitly: "Insufficient data to assess this claim"
- Use present tense for ongoing situations, past tense for confirmed events
- Avoid passive voice when attributing claims: "Iran claims X" not "it has been reported that X"
